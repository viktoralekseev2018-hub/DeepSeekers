# Автоматическое выделение самостоятельных услуг и генерация черновиков объявлений для Авито

## Описание проекта

Система автоматически определяет в тексте объявления о ремонте и отделке:
- какие микрокатегории услуг представлены в тексте объявления
- нужно ли создавать дополнительные черновики объявлений (сплитовать)
- если да, то для каких микрокатегорий
- предлагает текст для каждого черновика

## Микрокатегории

## Метрики качества

- **Precision (micro)** — доля корректно предложенных микрокатегорий среди всех предложенных
- **Recall (micro)** — доля корректно найденных микрокатегорий среди всех эталонных
- **F1-score (micro)** — основная метрика качества по дополнительным микрокатегориям
- **Accuracy по shouldSplit** — доля объявлений, для которых правильно определено, нужно ли создавать дополнительные черновики

## Формат входных данных

Входное объявление задается следующими полями:
- `itemId` — уникальный идентификатор объявления
- `sourceMcId` — идентификатор исходной микрокатегории объявления
- `sourceMcTitle` — название исходной микрокатегории объявления
- `description` — текст объявления

## Формат выходных данных

Результат работы решения содержит:
- `detectedMcIds` — микрокатегории, услуги из которых найдены в объявлении
- `shouldSplit` — признак необходимости создания дополнительных черновиков
- `drafts` — список предложенных черновиков

Каждый черновик содержит:
- `mcId` — идентификатор микрокатегории
- `mcTitle` — название микрокатегории
- `text` — предложенный текст черновика

## Алгоритм работы

1. **Детекция микрокатегорий**: Поиск ключевых фраз в тексте объявления с учетом приоритетов и контекста
2. **Определение сплита**: Анализ индикаторов предложения услуг отдельно vs в составе комплекса
3. **Генерация черновиков**: Извлечение релевантных фрагментов текста для каждой микрокатегории
